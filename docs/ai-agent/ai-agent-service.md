# AI Agent — `services/ai-agent` Implementation

The `services/ai-agent` service is a Python service responsible for executing agent conversations using the [OpenHands Software Agent SDK](https://docs.openhands.dev/sdk). It consumes trigger events from a Valkey Stream, manages Docker container lifecycles, and streams conversation events back.

## Valkey Stream Protocol

### Inbound stream: `paca:agent:triggers`

The `services/api` publishes to this stream. The ai-agent service uses a consumer group (`ai-agent-workers`) so that multiple service replicas do not double-process events.

**Message fields:**

| Field | Type | Description |
|---|---|---|
| `id` | string | Message ID (generated by Valkey) |
| `trigger_type` | string | `task_assigned` \| `comment_mention` \| `chat_message` \| `description_write` |
| `conversation_id` | UUID | Pre-allocated by the API, used as the OpenHands conversation ID |
| `agent_id` | UUID | |
| `project_id` | UUID | |
| `task_id` | UUID? | Present for `task_assigned`, `comment_mention`, and `description_write` |
| `comment_id` | UUID? | Present for `comment_mention` |
| `chat_session_id` | UUID? | Present for `chat_message` |
| `message` | string | User message text (all trigger types) |
| `actor_member_id` | UUID | Project member who triggered the agent |
| `repo_plugin_ids` | string | Comma-separated list of repo-plugin IDs available to this trigger |

### Outbound stream: `paca:agent:events`

The ai-agent service publishes events here as conversations progress. The `services/realtime` service relays these to Socket.IO clients.

**Message fields:**

| Field | Type | Description |
|---|---|---|
| `conversation_id` | UUID | |
| `project_id` | UUID | |
| `event_type` | string | OpenHands SDK event class name, e.g. `MessageAction`, `CmdRunAction`, `AgentFinishAction` |
| `event_source` | string | `agent` \| `user` \| `system` |
| `event_index` | integer | Sequential index within the conversation |
| `payload` | JSON string | Full serialized OpenHands event |
| `status` | string | Current conversation status after this event |

The API service also subscribes to this stream to persist events to `agent_conversation_events` and write replies to tasks/comments when a `AgentFinishAction` is received.

## Conversation Execution

The core execution flow in `executor.py`:

```python
from openhands.sdk import LLM, Agent, AgentContext, Conversation
from openhands.sdk.context import Skill, KeywordTrigger
from openhands.workspace import DockerWorkspace
from openhands.sdk.secret import SecretSource

async def run_conversation(trigger: TriggerEvent, agent_config: AgentConfig):
    # 1. Build LLM config
    llm = LLM(
        model=f"{agent_config.llm_provider}/{agent_config.llm_model}",
        api_key=SecretStr(await secrets.get(agent_config.llm_api_key_secret_ref)),
        base_url=agent_config.llm_base_url or None,
    )

    # 2. Build skills
    skills = build_skills(agent_config.skills)

    # 3. Build MCP config
    mcp_config = build_mcp_config(agent_config.mcp_servers)

    # 4. Build agent context — the system suffix is the base system prompt
    # plus trigger-specific metadata (action type, IDs, repo setup) from
    # build_trigger_suffix(trigger, all_repos_info); build_initial_prompt(trigger)
    # builds only the first user message, independent of repo info.
    system_suffix = agent_config.system_prompt + build_trigger_suffix(trigger, all_repos_info)
    agent_context = AgentContext(
        skills=skills,
        system_message_suffix=system_suffix,
    )

    # 5. Create agent
    agent = Agent(
        llm=llm,
        tools=get_default_tools(),
        mcp_config=mcp_config,
        agent_context=agent_context,
    )

    # 6. Spin up Docker workspace
    with DockerWorkspace(
        server_image="ghcr.io/paca-ai/paca-agent-server:latest",
        host_port=allocate_port(),
        platform=detect_platform(),
    ) as workspace:

        # 7. Clone repository if needed (trigger.repo_plugin_ids is a list —
        # a project can have more than one repository plugin installed)
        if trigger.repo_plugin_ids:
            token_source = RepoTokenSecretSource(trigger.repo_plugin_ids[0], trigger.project_id)
            workspace.execute_command(
                f"git clone https://x-access-token:$GIT_TOKEN@{trigger.repo_clone_url} /workspace/repo"
            )

        # 8. Create conversation
        conversation = Conversation(
            agent=agent,
            workspace=workspace,
            conversation_id=trigger.conversation_id,
            persistence_dir=f"/data/conversations/{trigger.agent_id}",
            callbacks=[make_event_callback(trigger, streams)],
        )

        if trigger.repo_plugin_ids:
            conversation.update_secrets({"GIT_TOKEN": token_source})

        # 9. Send initial message — build_initial_prompt falls back to a
        # default for empty-message triggers (e.g. a human task assignment)
        conversation.send_message(build_initial_prompt(trigger))

        # 10. Run (blocking; managed by thread pool in the worker)
        conversation.run(max_iterations=agent_config.max_iterations)

        # 11. Post-process: create PR, write reply
        await post_process(trigger, conversation, workspace)
```

### Event Callback

Every OpenHands SDK event is forwarded to the Valkey stream in real time:

```python
def make_event_callback(trigger, streams):
    idx = itertools.count()
    def callback(event):
        streams.publish("paca:agent:events", {
            "conversation_id": str(trigger.conversation_id),
            "project_id": str(trigger.project_id),
            "event_type": type(event).__name__,
            "event_source": getattr(event, "source", "agent"),
            "event_index": next(idx),
            "payload": event.model_dump_json(),
            "status": "running",
        })
    return callback
```

## Docker Container Strategy

- Each conversation gets a **dedicated Docker container** using `DockerWorkspace`.
- The Paca `agent-server` image (`ghcr.io/paca-ai/paca-agent-server:latest`, a thin wrapper around OpenHands' `agent-server` with the Paca MCP server pre-installed) is used by default and can be overridden per agent type.
- The container is automatically stopped and removed when the `DockerWorkspace` context manager exits (normal finish, stop, or unhandled error).
- Container network access is restricted: containers are placed on an isolated Docker network that has no route to other Paca service containers.
- Port allocation: a small port pool (`10000–10999`) is managed by the ai-agent service. Each active conversation claims one port; it is released when the conversation ends.
- **Container resource limits** are configured at the `DockerWorkspace` level:
  - CPU: 2 cores
  - Memory: 4 GB
  - Disk: ephemeral volume capped at 10 GB

## Repository Access

See [repository-plugin-adapter.md](repository-plugin-adapter.md) for the full protocol. Summary:

```python
class RepoTokenSecretSource(SecretSource):
    """Dynamically fetches a short-lived VCS token from the repository plugin."""

    def __init__(self, plugin_id: UUID, project_id: UUID):
        self.plugin_id = plugin_id
        self.project_id = project_id
        self._cached_token: str | None = None
        self._expires_at: float = 0

    def get_value(self) -> str:
        if time.time() < self._expires_at - 60:
            return self._cached_token
        # Fetch fresh token from plugin API (internal HTTP call)
        response = httpx.get(
            f"{settings.api_base_url}/internal/plugins/{self.plugin_id}/repo-token",
            headers={"X-Internal-Key": settings.internal_api_key},
            params={"project_id": str(self.project_id)},
        )
        data = response.json()
        self._cached_token = data["token"]
        self._expires_at = data["expires_at"]
        return self._cached_token
```

The token is passed as `SecretSource` so its value is **masked in all OpenHands event output** and never stored in conversation state.

## Skills & MCP Server Injection

### Skills

Agent skills stored in `agent_skills` are loaded at conversation start:

```python
def build_skills(db_skills: list[AgentSkillRow]) -> list[Skill]:
    result = []
    for row in db_skills:
        if not row.is_enabled:
            continue
        trigger = None
        if row.triggers:
            trigger = KeywordTrigger(keywords=row.triggers)
        # For inline skills, content is stored in DB
        # For marketplace/github skills, content is fetched and cached
        content = resolve_skill_content(row)
        result.append(Skill(
            name=row.skill_name,
            content=content,
            trigger=trigger,
        ))
    return result
```

### MCP Servers

```python
def build_mcp_config(db_servers: list[AgentMCPServerRow]) -> dict:
    servers = {}
    for row in db_servers:
        if not row.is_enabled:
            continue
        if row.transport == "stdio":
            servers[row.server_name] = {
                "command": row.command,
                "args": row.args,
                "env": row.env,
            }
        else:
            servers[row.server_name] = {
                "url": row.url,
                **({ "auth": "oauth" } if row.transport == "oauth" else {}),
            }
    return {"mcpServers": servers}
```

## Pause / Resume / Stop

The REST API endpoints delegate to the active conversation object, which is stored in an in-memory registry keyed by `conversation_id`:

```python
# In-memory registry (per process; multi-replica note below)
active_conversations: dict[UUID, Conversation] = {}

@router.post("/conversations/{id}/pause")
async def pause_conversation(id: UUID):
    conv = active_conversations.get(id)
    if conv is None:
        raise HTTPException(404)
    conv.pause()
    await db.update_conversation_status(id, "paused")
    return {"status": "paused"}

@router.post("/conversations/{id}/resume")
async def resume_conversation(id: UUID):
    conv = active_conversations.get(id)
    if conv is None:
        # Restore from persistence and resume on this worker
        conv = await restore_conversation(id)
    asyncio.create_task(run_in_thread(conv.run))
    await db.update_conversation_status(id, "running")
    return {"status": "running"}

@router.post("/conversations/{id}/stop")
async def stop_conversation(id: UUID):
    conv = active_conversations.pop(id, None)
    if conv:
        conv.close()
    await db.update_conversation_status(id, "stopped")
    return {"status": "stopped"}
```

> **Note on multi-replica deployments**: In a multi-replica setup, pause/resume/stop requests must be routed to the replica holding the active conversation. Use a Valkey key (`paca:agent:active:{conversation_id}`) to track which replica owns a conversation, and proxy the control request to that replica. Alternatively, use a single-replica model with larger resource limits.

## REST API

Base path: `http://ai-agent:8080/`

| Method | Path | Description |
|---|---|---|
| `GET` | `/health` | Liveness check |
| `GET` | `/conversations/:id` | Get conversation metadata and status |
| `GET` | `/conversations/:id/events` | Get all persisted events (paginated) |
| `GET` | `/conversations/:id/events/stream` | SSE stream of live events (proxied by `services/api` for web clients) |
| `POST` | `/conversations/:id/pause` | Pause a running conversation |
| `POST` | `/conversations/:id/resume` | Resume a paused conversation |
| `POST` | `/conversations/:id/stop` | Permanently stop and destroy the conversation |
| `POST` | `/conversations/:id/message` | Send an additional message to a running/paused conversation |

All endpoints are **internal-only** — not exposed through the public API gateway. `services/api` acts as the public-facing proxy.
